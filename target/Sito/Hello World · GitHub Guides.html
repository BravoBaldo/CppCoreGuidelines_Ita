Che cos'è GitHub?

GitHub è una piattaforma che ospita del codice per controllarne le versioni e gestire la collaborazione. Consente a più persone di collaborare assieme ad un progetto da qualsiasi posto.

Questo tutorial insegna gli elementi essenziali di GitHub come i repository, i branch, i commit, e le Pull Request. Si creerà il proprio repository di "Hello World" e si vedrà il flusso di lavoro della "Pull Request", un modo popolare per creare e revisionare il codice.

Non è necessaria alcuna codifica

Per completare questo tutorial, c'è bisogno di un account su GitHub.com e l'accesso ad Internet. Non è necessario saper programmare, usare una riga di comando né installare Git (la versione 'stand-alone' del [version control software GitHub]).

Suggerimento: Aprire questa guida in una finestra del browser separata (o un altro tab) in modo da poterla seguire mentre si seguono i passi del tutorial.

Passo 1. Creare un Repository

Un repository viene solitamente usato per organizzare un singolo progetto. I repository possono contenere cartelle e file, immagini, video, fogli di calcolo e dati – tutto il necessario per il progetto. Si raccomanda di includere un README (LEGGIMI) o un file con le informazioni sul proprio progetto. GitHub ne propone l'aggiunta quando si crea un nuovo repository. Propone anche altre comuni possibilità come il file della licenza.

Il repository hello-world può diventare un luogo dove riporre idee, risorse o anche dove condividere e discutere cose con altri.

Per creare un nuovo repository

  1. Nell'angolo in alto a destra, di fianco al proprio avatar o identicon, si clicca su e si seleziona poi New repository.
  2. Chiamare il repository hello-world.
  3. Scriverne una breve descrizione.
  4. Selezionare Initialize this repository with a README.

Cliccare su Create repository.

Passo 2. Creare un Branch

Il Branching è il metodo per lavorare su diverse versioni di un repository contemporaneamente.

Per default il repository ha un branch chiamato master che è considerato essere il branch definitivo. I branch si usano per sperimentare e fare correzioni prima di confermarle [commit] nel master.

Quando si crea un branch diverso da master, si sta facendo una copia, o una foto, di master così com'è in quel momento. Se qualcun altro ha fatto delle modifiche al branch master mentre si lavorava sul proprio branch, si possono prelevare quegli aggiornamenti.

Questo diagramma mostra:

  • Il branch master
  • Un nuovo branch chiamato feature (perché si stanno facendo delle ‘feature’ su questo branch)
  • Il viaggio che intraprende la feature prima che sia inclusa [merge] nel master

Avete mai salvato diverse versioni di un file? Qualcosa come:

  • storia.txt
  • storia-joe-modifica.txt
  • storia-joe-modifica-corretta.txt

I branch assolvono alla stessa funzione nei repository GitHub.

Qui in GitHub, i nostri sviluppatori, scrittori e progettisti usano i branch per tenere sia traccia delle correzioni che le funzionalità, separate dal branch master (di produzione). Quando una modifica è pronta, viene inserita [merge] dal suo branch al master.

Per creare un nuovo branch

  1. Si va nel proprio nuovo repository hello-world.
  2. Si clicca sul menù a discesa nella parte superiore dell'elenco dei file con la dicitura branch: master.
  3. Si digita un nome per un branch, readme-edits, nella casella del nuovo branch.
  4. Si seleziona la casella blu Create branch o si preme “Enter” sulla tastiera.

Ora si hanno due branch, master e readme-edits. Appaiono esattamente gli stessi, ma non per molto! In seguito si aggiungeranno modifiche al nuovo branch.

Passo 3. Fare modifiche ed eseguirne il commit

Bravo! Ora, si è nella visualizzazione del codice per il branch readme-edits, che è una copia di master. Si fa qualche modifica.

Su GitHub, le modifiche salvate sono chiamate commit. Ciascun commit ha un commit message associato, che è una descrizione del motivo della particolare modifica. I messaggi dei commit formano la storia delle modifiche, in modo che altri collaboratori possano capire cosa sia stato fatto e perché.

Fare modifiche ed eseguirne il commit

  1. Cliccare sul file README.md.
  2. Cliccare sull'icona della matita nell'angolo in alto a destra della vista del file per modificarlo.
  3. Nell'editor, si scrive qualcosa su se stessi.
  4. Scrivere un messaggio per il commit che descriva le modifiche.
  5. Cliccare sul pulsante Commit changes.

Queste modifiche verranno apportate solo al file README del branch readme-edits, ora, quindi, questo branch contiene cose diverse da quello di master.

Passo 4. Aprire una "Pull Request"

Belle modifiche! Ora che si hanno delle modifiche in un branch diverso da master, si può aprire una pull request.

Le "Pull Request" costituiscono il cuore della collaborazione su GitHub. Quando si apre una pull request, si stanno proponendo le proprie modifiche e si sta richiedendo a qualcuno di rivederle, condividere il proprio contributo e inserirlo nel suo branch. Le "Pull request" mostrano le diffs, o differenze, tra i contenuti di entrambi i branch. Le modifiche, le aggiunte e le rimozioni appaiono in verde e in rosso.

Non appena si effettua un commit, è possibile aprire una "pull request" e comincia una discussione, anche prima che il codice sia finito.

Utilizzando il @mention system di GitHub nel proprio messaggio di "pull request", si può richiedere un feedback da persone o da team specifici, sia che si trovino nella stessa stanza o che distino 10 fusi orari.

Si possono anche aprire delle "pull request" nel proprio repository ed eseguirne da soli un "merge". È un ottimo modo per imparare il flusso di GitHub prima di lavorare su progetti più grandi.

Cliccare sul tab Pull Request, poi dalla pagina della "Pull Request", cliccare sul pulsante verde New pull request.
Nella casella Example Comparisons, selezionare il branch creato, readme-edits, da confrontare con master (l'originale).
Controllare le modifiche nelle [diffs] dalla pagina Compare, verificando che siano quelle che si vogliono confermare.
Una volta soddisfatti delle modifiche da inviare, si clicca sul grande pulsante verde Create Pull Request.
Dare un titolo alla "pull request" e scrivere una breve descrizione delle modifiche.

Al termine del messaggio, cliccare su Create pull request!

Suggerimento: Si possono usare le emoji e il "drag and drop" di immagini e gif nei commenti e nelle "Pull Request".

Passo 5. Eseguire il "Merge" della propria "Pull Request"

In questo ultimo passaggio, è tempo di riunire le modifiche – unendo il proprio branch readme-edits al branch master.

  1. Cliccare sul pulsante verde Merge pull request per inserire le modifiche in master.
  2. Cliccare su Confirm merge.
  3. Si va avanti e si cancella il branch, dato che le modifiche sono state incorporate, col pulsante Delete branch nella casella viola.

Festeggiamo!

Completando questo tutorial, si è imparato a creare un progetto e fare una "pull request" su GitHub!

Ecco cosa si è realizzato in questo tutorial:

  • Creato un repository open source
  • Avviato e gestito un nuovo branch
  • Modificato un file ed eseguito il commit di tali modifiche in GitHub
  • Aperta e inserita [merge] una "Pull Request"

Si dia un'occhiata al proprio profilo GitHub e si vedranno le nuove contribution squares!

Per ulteriori informazioni sulla potenza delle "Pull Request", si raccomanda la lettura della GitHub flow Guide. Si può anche visitare GitHub Explore e farsi coinvolgere in un progetto Open Source.
